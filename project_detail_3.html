[프로젝트 제목: Zero Trust 기반 클라우드 망분리 구축]

클라우드 환경에서 레거시 망분리 개념을 탈피하고, 최소 권한 원칙에 기반한 Zero Trust Network Architecture(ZTNA)를 구현한 프로젝트입니다.

기간: 2024.03 - 2024.06 (4개월)
역할: 보안 아키텍트 (100% 기여)
결과: 내부 보안 지표 15% 개선 달성
#ZeroTrust #AWS #Microsegmentation #Terraform

01. 문제 정의 및 프로젝트 목표

기존 클라우드 환경은 넓은 범위의 네트워크 허용 규칙으로 인해 측면 이동(Lateral Movement) 공격에 취약했습니다. 내부 사용자 또는 워크로드 간의 통제 부재는 침해 사고 시 피해 확산을 막기 어렵다는 심각한 문제를 내포하고 있었습니다.

  • **문제:** 과도한 네트워크 접근 허용으로 인한 측면 이동 공격 위험.
  • **목표 1:** 모든 통신에 대한 인증/인가(Authentication/Authorization) 강제화.
  • **목표 2:** 워크로드 간의 Microsegmentation을 구현하여 최소 권한 통신 모델 정착.
  • **목표 3:** 모든 보안 정책을 코드(IaC)로 관리하여 배포 자동화 및 관리의 용이성 확보.

02. 프로젝트 방법론 (Methodology)

Google BeyondCorp 모델을 기반으로 AWS 환경에 적합한 ZTNA 모델을 설계했습니다. AWS Network Firewall과 Security Groups, 그리고 Terraform을 핵심 툴로 사용했습니다.

사용 기술 및 툴

AWS Network Firewall Terraform Lambda Python

Terraform을 이용한 보안 그룹 자동화 예시


resource "aws_security_group" "web_sg" {
  name        = "web-server-sg"
  vpc_id      = var.vpc_id

  # 인바운드: 오직 로드밸런서(LB)로부터의 443 포트만 허용
  ingress {
    from_port   = 443
    to_port     = 443
    protocol    = "tcp"
    security_groups = [aws_security_group.lb_sg.id]
  }

  # 아웃바운드: 오직 데이터베이스(DB)로의 3306 포트만 허용
  egress {
    from_port   = 3306
    to_port     = 3306
    protocol    = "tcp"
    security_groups = [aws_security_group.db_sg.id]
  }
}

03. 핵심 성과 및 Visuals

프로젝트 완료 후, 네트워크 세그멘테이션 평가 지표가 크게 개선되었으며, 침해사고 대응팀의 내부 이동 경로 분석 시간이 단축되었습니다.

또한, 보안 그룹 정책의 형상 관리가 용이해져, 새로운 서비스를 배포할 때 보안 정책 승인 시간이 1/2로 단축되는 효과를 가져왔습니다.

04. 결론 및 Learning

Zero Trust 개념을 클라우드 환경에 적용하는 과정에서 기술적 난이도와 조직 내의 문화적 변화가 가장 큰 도전이었습니다. 그러나 코드 기반 인프라(IaC)를 통해 보안 정책을 관리함으로써, 보안과 민첩성을 동시에 확보할 수 있음을 입증했습니다.

  • **Learning 1:** 보안 정책을 코드로 관리할 때, 정책의 DRY(Don't Repeat Yourself) 원칙 적용이 매우 중요함.
  • **Learning 2:** 클라우드 네이티브 보안 솔루션 활용이 레거시 솔루션보다 유연하고 비용 효율적임.
